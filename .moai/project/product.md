# Product Context — 한국 주식 5-페르소나 AI 매매 시스템

## 한 줄 정의

박세훈 개인이 자기자본 1,000만원으로, 매크로/마이크로/결정/리스크/포트폴리오/회고 5개 LLM 페르소나가 협업·견제하여 한국 주식 매매를 수행하는 학습·운영 시스템. 모의투자에서는 완전 자동, 실거래 진입은 별도 결정.

## 왜 이 제품인가 (Why)

### 사용자 컨텍스트

박세훈 (50대, 부산 거주, 자녀 2명 부양). 공작기계 제어설계·커스텀 SW 개발 15년 + AI 거버넌스·기획·개발 5년 경력. 시니어 책임급 직전 연봉 1.4억. 한국 시니어 재취업 한계로 부업/자본운용 경로 모색 중.

### 동기

- 본업 대체가 아닌 **자본 운용 + AI 거버넌스 실증**
- AI 거버넌스 5년 경력자로서 *"AI에게 의사결정을 위임할 때 어떤 거버넌스가 필요한가"* 를 자기 자본으로 직접 검증
- 가족 부양 책임 → **자본 보전이 수익률보다 우선** (모의 단계에서 자본 위험 0)
- 향후 자본 증액 시 **그대로 사용 가능한 인프라**

### 본 제품의 입장

대부분의 "AI 트레이딩"은 단일 LLM에게 매매를 시키며 LLM의 시계열 예측 능력을 과장한다. 본 제품은 다음을 채택한다:

- **5-페르소나 분리** — 단일 LLM이 모든 일을 하지 않음
- **결정 페르소나 = 박세훈 페르소나** — 박세훈 본인의 트레이딩 철학·위험 성향이 시스템 프롬프트로 명시화
- **리스크 페르소나로 SoD(Separation of Duties) 적용** — 결정자와 검증자를 분리, AI 거버넌스 정석
- **모든 페르소나 응답·근거·토큰비용 영구 기록** — 사후 검증·학습 자산
- **위험 한도는 AI도 못 넘음** — 회로차단기, 한도 5종은 페르소나 결정과 독립
- **모의 6개월 검증 후 실거래 별도 결정** — 페르소나 일관성·성과·룰 기반 벤치마크 비교

### 솔직히 인정하는 한계 (NFR로 박힘)

- LLM은 시계열 예측을 못 한다 — 페르소나는 "분석"을 하지 "예측"을 하지 않음
- 같은 입력에 같은 답을 안 한다 — 결정 근거를 매번 영구 기록하여 일관성 검증
- 학습 데이터 lookahead bias 위험 — 백테스트 대신 forward 모의 실거래로만 검증
- 환각 가능성 — 데이터 소스 자체의 신뢰성에 의존, 페르소나 간 교차 검증으로 부분 완화

| **시계열 브리핑** | **모든 페르소나 호출 직후 텔레그램에 의사결정 내용·근거·자산현황 즉시 브리핑** — 박세훈 님이 *"AI가 지금 뭘 하는지"* 실시간 추적 |
| 거버넌스 실증 | AI에게 매매를 위임할 때 SoD·감사로그·점진적 권한 부여 패턴을 자기 자본으로 검증 |

| 학습 자산 | 자본 1,000만원 → 1억 → 3억으로 증액해도 같은 시스템 사용 |

## 5-페르소나 정의

### 1. 매크로 페르소나 (Macro)
- **모델**: Claude Opus 4.7 (`claude-opus-4-7`)
- **주기**: 주 1회, 금요일 장 마감 후 18:00 KST
- **분석**:
  - 거시경제 (Fed/한은 금리, FOMC, GDP, CPI/PPI, 실업률, USD/KRW, 국채 수익률)
  - 글로벌 자산 흐름 (S&P500, Nasdaq, 미국 섹터 ETF, 금/원자재, BTC, VIX)
  - 한국 시장 자체 흐름 (KOSPI/KOSDAQ, 외국인/기관/개인 매매, 신용잔고, 거래대금)
  - 정책/이벤트 (FOMC 일정, 한은 금통위, 어닝 시즌, 주요국 선거)
- **출력**: 향후 1주 시장 모멘텀(상승/중립/하락) + 위험선호도(risk-on/off) + 워치리스트
- **하류 페르소나에 미치는 영향**: 결정 페르소나가 캐시된 최신 응답 참조

### 2. 마이크로 페르소나 (Micro)
- **모델**: Claude Sonnet 4.6 (`claude-sonnet-4-6`)
- **주기**: 매 영업일 장 마감 후 18:00 KST
- **분석**:
  - 공시·뉴스 (DART 공시, 종목 뉴스, 실적, 컨센서스, 자사주, 배당)
  - 재무·밸류에이션 (PER, PBR, ROE, 매출/영업이익 성장률, 부채비율, 동종 비교)
  - 기술적 지표 (이동평균, RSI, MACD, 볼린저, 거래량)
  - 수급·차트 패턴 (외국인/기관 보유 변화, 거래량 이상, 공매도 잔고)
- **유니버스**: KOSPI 200 + KOSDAQ 150 + 사용자 워치리스트
- **출력**: 매수/매도/관망 후보 종목 리스트 + 각 종목 근거

### 3. 결정 페르소나 (Decision = 박세훈 페르소나)
- **모델**: Claude Sonnet 4.6
- **주기**: 매 영업일 18:30 KST
- **시스템 프롬프트 핵심** (M4 구현 시 박세훈과 정밀화):
  - 자본 보전이 수익률보다 우선 (가족 부양)
  - AI 거버넌스 관점의 신중함 — 환상 거부
  - 공작기계 도메인 출신 — 시스템적/엔지니어링적 사고
  - 위험 한도 절대 위반 안 함
  - 페르소나 의견 충돌 시 보류
- **입력**: 매크로 페르소나 최신 응답(주간 캐시) + 마이크로 페르소나 당일 응답
- **출력**: 매매 시그널 (종목/방향/수량/근거)

### 4. 리스크 페르소나 (Risk Officer) ← AI 거버넌스 핵심
- **모델**: Claude Sonnet 4.6
- **주기**: 결정 페르소나가 매매 시그널을 낼 때마다 (이벤트 트리거)
- **역할**: 결정 페르소나의 매매 제안을 **독립적으로** 검증
  - 자본 집중도, 섹터 편중, 보유 종목과 상관관계
  - 최악 시나리오 손실(downside) 추정
  - 한도 5종 사후 검증
  - 페르소나 응답의 내부 모순 탐지
- **출력**: APPROVE / HOLD / REJECT + 근거
- **거버넌스 의미**: SoD — 의사결정자(결정 페르소나)와 검증자(리스크 페르소나)의 분리. 회로차단기는 *코드 룰*, 리스크 페르소나는 *의미 단위* 리스크. 두 레이어로 자본 보전.

### 5. 포트폴리오 페르소나 (Portfolio Manager) — M5에서 추가
- **모델**: Claude Sonnet 4.6
- **주기**: 매 영업일 18:35 KST (결정 페르소나 직후, 리스크 페르소나 직전)
- **활성화 조건**: 보유 종목 5개 이상 시 자동 활성. 그 전엔 비활성.
- **역할**:
  - 결정 페르소나의 매매 시그널을 **포트폴리오 차원에서 조정**
  - 사이즈 결정 (몇 주? 자본의 몇 %?)
  - 섹터 분산, 상관관계 검토
  - 신규 매수 vs 기존 종목 리밸런스 제안
- **출력**: 조정된 매매 시그널 (수량 조정 또는 reject)

### 6. 회고 페르소나 (Retrospective) — M5에서 추가
- **모델**: Claude Sonnet 4.6
- **주기**: 주간, 일요일 임의 시각
- **역할**:
  - 지난 주 매매 결과 + 페르소나 응답 종합 분석
  - *"매크로가 강세장이라 했는데 시장이 빠졌다 → 다음 주 가중치 어떻게?"* 같은 메타-학습
  - 페르소나 시스템 자체의 개선 제안 (시스템 프롬프트 보완 후보)
- **출력**: 주간 회고 리포트 (텔레그램 발송) + 페르소나 시스템 개선 제안

## 단계별 출시

본 SPEC(SPEC-TRADING-001) 구현 범위는 **M1~M5 (모의 자동 매매)** 까지. M6 실전 투자는 3주 모의 결과 평가 후 별도 SPEC으로 작성.

1. **M1 인프라** — Docker compose + Postgres + .env 보안 + healthcheck + backup.sh
2. **M2 KIS API 연결(모의)** — auth/market/account/order, DB 스키마 v1, 모의 1주 매수 검증
3. **M3 데이터·벤치마크** — 데이터 소스(pykrx/yfinance/FRED·ECOS/DART) 통합 + 룰 기반 벤치마크 전략(SMA, 듀얼 모멘텀) 백테스트 (2019-01-01 이후)
4. **M4 5-페르소나 장중 매매 시스템** — Macro(주간 Opus) + Micro(Pre-market Sonnet) + Decision(Pre-market+장중4회+이벤트, Sonnet) + Risk(이벤트, Sonnet) + 텔레그램 시계열 브리핑 + 모의 자동 매매
5. **M5 위험관리·관측성·페르소나 확장** — 회로차단기, 한도 5종, /halt /resume, 페르소나 응답 영구 기록, 16:00 일일 리포트, **Portfolio Persona** + **Retrospective Persona** 추가, 알림 피로 침묵 모드, 3주 모의 운영

### M6 (Future Scope, 본 SPEC 범위 외)

**M6 실전 투자**는 본 SPEC에 구현되지 않음. M5 완료 후 3주 모의 운영 결과를 평가하고 별도 SPEC(예: SPEC-TRADING-002)으로 진행. 후보 시나리오:
- (a) 실거래 진입 — 자본 비중 5~30% 범위 (정확한 % 는 3주 결과 기반 결정)
- (b) 모의 연장 (1~3개월 추가)
- (c) 시스템 보완 후 재시도

본 SPEC에서 M6 관련 유일한 요구사항은 *"`live_unlocked` 플래그가 false 상태에서 실거래 차단"* 안전장치 뿐.

## 성공 정의

**3주 모의 운영의 검증 한계 인식**: 3주(영업일 ~15일)는 시스템 동작 검증엔 충분하나 페르소나 성과의 통계적 유의미성을 확보하기엔 표본이 부족하다. 따라서 정량 지표보다 정성 지표를 우선한다.

M5 완료 후 **모의 3주** 운영 시 충족되어야 할 항목:
- 시스템 무중단 가동 (컨테이너 재시작 ≤ 2회)
- 모든 페르소나 응답·매매·리스크 검토 100% DB 기록
- 회로차단기·긴급 정지가 한 번도 우회되지 않음
- /halt /resume 명령어 정상 작동 (검증 이벤트 audit_log 기록)
- 백업 → 새 환경 복원 리허설 1회 통과
- **리스크 페르소나가 결정 페르소나를 의미 있게 견제** (REJECT/HOLD ≠ 0 — 0이면 SoD 실패)
- 페르소나 응답 일관성 측정 가능 (매크로/마이크로/결정/리스크 응답이 사후 검증 가능)

**M6 실거래 진입 조건은 3주 모의 결과 검토 후 확정**한다. 가능 시나리오:
- (a) 진입 — 자본 비중 5~30% 범위에서 결정
- (b) 모의 연장 (1~3개월 추가)
- (c) 시스템 보완 후 재시도

박세훈이 *"AI 거버넌스 관점에서 이 시스템의 한계와 가능성을 정확히 안다"* 라고 판단할 때 성공.

자본 1,000만원의 **3~5%** — 부담 큼. 자본 1억 이상 증액 시 0.3~0.5%로 떨어져 적정.

비용 절감 옵션 (M5+ 운영 후 검토):
- 장중 정기 4회 → 2회로 축소 → 월 ~7만원 절감
- 이벤트 트리거 임계치 ±3% → ±5% 완화
- 매크로 주간 → 격주 (시장 안정기)
